## 1. 项目概述

### 1.1 目标

为现有 ERP 系统开发 Android 手机端应用，提供完整的移动端 ERP 体验。

### 1.2 使用场景

- 销售开单
- 库存管理
- 商品查询
- 扫码入库
- 财务收款
- 报表查看

---

## 2. 技术选型

### 2.1 技术栈

| 项目 | 选择 | 理由 |
|------|------|------|
| **框架** | uni-app + Vue 3 | 与现有 Web 前端技术栈一致 |
| **UI 组件** | uView UI 3.0 | 成熟的移动端组件库 |
| **状态管理** | Pinia | Vue 3 官方推荐 |
| **后端** | 复用现有 ERP API | 无需重复开发 |
| **打包** | Android APK | 目标平台仅 Android |

### 2.2 原生功能

| 功能 | API |
|------|-----|
| **相机扫码** | uni.scanCode |
| **指纹/面容** | uni.startSoterAuthentication |

## 5. 核心功能设计

### 5.1 扫码入库

**支持条码类型**：
- EAN-13 / EAN-8（商品条码）
- Code128（物流条码）
- QR Code（二维码）

**页面设计**：
```
┌─────────────────────────────────┐
│  ←  扫码入库                     │
├─────────────────────────────────┤
│    ┌───────────────────────┐   │
│    │    📷 扫描区域         │   │
│    └───────────────────────┘   │
│    手动输入: [            ] 🔍  │
├─────────────────────────────────┤
│  已扫描 (2)                      │
│  ┌───────────────────────────┐ │
│  │ 商品A          + 10       │ │
│  │ 商品B          + 20       │ │
│  └───────────────────────────┘ │
│        [ 确认入库 ]             │
└─────────────────────────────────┘
```

**交互**：
- 扫码成功 → 自动识别商品，数量 +1
- 点击商品行 → 可修改数量/删除
- 长按商品行 → 查看商品详情
- 连续扫码 → 自动累加数量

### 5.2 销售开单 + 收款

**完整流程**：
```
开单 → 添加商品 → 购物车确认 → 选择客户 → 提交订单 → 收款 → 完成
```

**收款页面**：
```
┌─────────────────────────────────┐
│  ←  收款                         │
├─────────────────────────────────┤
│  订单金额：¥ 580.00              │
├─────────────────────────────────┤
│  支付方式                        │
│  ┌─────┐ ┌─────┐ ┌─────┐       │
│  │ 微信 │ │支付宝│ │ 现金 │       │
│  │  ✓  │ │     │ │     │       │
│  └─────┘ └─────┘ └─────┘       │
├─────────────────────────────────┤
│    ┌───────────────────────┐   │
│    │    📱 商家收款码        │   │
│    │    [二维码图片]         │   │
│    └───────────────────────┘   │
│  客户扫码支付后点击确认          │
├─────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐      │
│  │ 挂账    │  │ 确认收款 │      │
│  └─────────┘  └─────────┘      │
└─────────────────────────────────┘
```

**支付方式**：
- 微信支付 → 显示微信收款码
- 支付宝 → 显示支付宝收款码
- 现金 → 直接确认收款
- 挂账 → 记入客户应收账款

### 5.3 收款码绑定

在设置页面上传/管理收款码图片。

## 9. 开发计划（TDD）

### 9.1 测试策略

| 测试类型 | 工具 | 覆盖范围 |
|----------|------|----------|
| **单元测试** | Vitest | 工具函数、状态管理、API 封装 |
| **组件测试** | Vitest + @vue/test-utils | UI 组件行为 |
| **E2E 测试** | Appium | 完整用户流程 |

### 9.2 开发阶段

| 阶段 | 测试内容 | 开发内容 | 预计时间 | 提交 |
|------|----------|----------|----------|------|
| **Phase 1** | API/Token/登录/首页测试 | 项目搭建 + 认证 + 首页 | 3-4 天 | Gitee |
| **Phase 2** | 商品/扫码测试 | 商品模块 + 扫码入库 | 4-5 天 | Gitee |
| **Phase 3** | 开单/收款测试 | 销售开单 + 收款流程 | 5-6 天 | Gitee |
| **Phase 4** | 库存测试 | 库存管理 + 入库出库 | 4-5 天 | Gitee |
| **Phase 5** | 客户/财务测试 | 客户 + 财务 + 报表 | 4-5 天 | Gitee |
| **Phase 6** | 集成测试 + E2E | 设置 + 生物识别 + 完整测试 | 3-4 天 | Gitee |

**总计：约 23-29 天**

### 9.3 TDD 流程

每个功能模块：
1. **先写测试用例**
2. **运行测试 → 失败（红灯）**
3. **编写最小代码**
4. **运行测试 → 通过（绿灯）**
5. **重构优化**

### 9.4 Git 分支策略

```
main (稳定版本)
  │
  ├── develop (开发分支)
  │     │
  │     ├── feature/auth (Phase 1) → 合并 → 提交 Gitee
  │     ├── feature/product (Phase 2) → 合并 → 提交 Gitee
  │     ├── feature/sale (Phase 3) → 合并 → 提交 Gitee
  │     ├── feature/inventory (Phase 4) → 合并 → 提交 Gitee
  │     ├── feature/finance (Phase 5) → 合并 → 提交 Gitee
  │     └── feature/settings (Phase 6) → 合并 → 提交 Gitee
  │
  └── release/v1.0.0 (发布版本)
```
