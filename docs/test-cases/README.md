| 方法 | 说明 | 适用场景 |

## 重点设计方法说明

### 正交试验法（销售开单模块）

销售开单模块使用了正交试验法设计测试用例，涉及5个因素：

| 因素 | 水平 |
|------|------|
| A - 客户类型 | 普通客户、VIP客户、现金客户 |
| B - 支付方式 | 现金支付、微信支付、支付宝 |
| C - 折扣类型 | 无折扣、百分比折扣、固定金额折扣 |
| D - 商品数量 | 单件商品、多件商品 |
| E - 结算方式 | 全款支付、部分付款 |

使用 L9 混合水平正交表，仅用9条测试用例覆盖了主要的因素组合，有效减少了测试用例数量同时保证了测试覆盖率。

### 判定表法（财务管理模块）

财务管理模块使用判定表法设计了应收账款状态流转测试：

| 条件组合 | 应收状态 |
|----------|----------|
| 已收款 = 0 | 未付款 |
| 0 < 已收款 < 应收金额 | 部分付款 |
| 已收款 = 应收金额 | 已付清 |
