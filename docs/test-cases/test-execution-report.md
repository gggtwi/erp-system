## 四、缺陷报告

### BUG-001: 商品价格允许为0或负数

| 项目 | 内容 |
|------|------|
| **缺陷编号** | BUG-001 |
| **缺陷标题** | 商品管理模块允许创建价格为0或负数的商品 |
| **严重程度** | 中等 |
| **优先级** | 高 |
| **发现用例** | TC-PROD-006, TC-PROD-007 |
| **重现步骤** | 1. 新增商品<br>2. 输入价格为0或负数<br>3. 点击保存 |
| **预期结果** | 系统应提示"价格必须大于0"或"价格不能为负数" |
| **实际结果** | 商品创建成功，未进行价格校验 |
| **影响范围** | 商品管理模块 |
| **建议修复** | 在后端增加价格校验逻辑，确保价格必须大于0 |

## 五、测试结论

### 5.1 总体评价

ERP 系统整体功能基本正常，核心业务流程（登录、销售、库存、报表）运行稳定。

### 5.2 主要发现

1. **优点**:
   - 登录认证安全处理良好，错误提示不暴露具体信息
   - 销售开单库存校验正确，防止超卖
   - 报表数据统计准确
   - 库存预警功能正常

2. **待改进**:
   - 商品价格校验缺失，需增加后端校验
   - 测试覆盖率有待提高

### 5.3 建议

1. 修复商品价格校验缺陷
2. 增加更多自动化测试用例
3. 完善API文档
