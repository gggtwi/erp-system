# 5. 财务管理模块测试用例

## 模块概述
财务管理模块负责应收账款管理、收款核销、欠款管理等功能，是ERP系统财务核算的核心。

## 测试设计方法
- **判定表法**: 用于应收账款状态流转和多条件组合逻辑
- **场景设计法**: 用于完整收款核销流程测试

---

## 5.1 判定表设计

### 应收账款状态判定表

应收账款状态由以下条件决定：
- C1: 应收金额 > 0
- C2: 已收款金额 = 0
- C3: 已收款金额 < 应收金额
- C4: 已收款金额 = 应收金额

| 规则 | R1 | R2 | R3 | R4 |
|------|----|----|----|----|
| C1: 应收金额 > 0 | Y | Y | Y | Y |
| C2: 已收款 = 0 | Y | N | N | N |
| C3: 已收款 < 应收 | N | Y | N | N |
| C4: 已收款 = 应收 | N | N | Y | Y |
| **状态** | 未付款 | 部分付款 | 已付清 | 已付清 |

### 收款核销判定表

| 规则 | 收款金额 | 应收状态 | 核销后状态 |
|------|----------|----------|------------|
| R1 | = 应收余额 | 未付款 | 已付清 |
| R2 | = 应收余额 | 部分付款 | 已付清 |
| R3 | < 应收余额 | 未付款 | 部分付款 |
| R4 | < 应收余额 | 部分付款 | 部分付款 |
| R5 | > 应收余额 | 任意 | 提示超额/自动转为预收款 |

## 测试要点总结

1. **状态流转**: 应收账款状态需根据收款情况正确流转
2. **金额准确**: 所有金额计算必须准确，无误差
3. **关联完整**: 收款记录与应收账款、销售单需正确关联
4. **权限控制**: 财务操作需有相应权限
5. **审计追溯**: 所有财务操作需有完整记录

### 状态流转图

```
[创建销售单(部分付款)]
        ↓
    [未付款] ──部分收款──→ [部分付款]
        │                      │
        └────全额收款──────────┴──→ [已付清]
```
