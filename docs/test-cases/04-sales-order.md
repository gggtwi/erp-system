# 4. 销售开单模块测试用例

## 模块概述
销售开单模块是ERP系统的核心业务模块，涉及客户选择、商品选择、数量价格、折扣、支付等多种因素的组合，是测试设计的重点模块。

## 测试设计方法
- **正交试验法**: 用于多因素多水平组合测试（重点使用）
- **场景设计法**: 用于完整销售流程测试

---

## 4.1 正交试验法设计

### 因素与水平确定

销售开单涉及以下关键因素：

| 因素 | 水平1 | 水平2 | 水平3 |
|------|-------|-------|-------|
| A - 客户类型 | 普通客户 | VIP客户 | 现金客户 |
| B - 支付方式 | 现金支付 | 微信支付 | 支付宝 |
| C - 折扣类型 | 无折扣 | 百分比折扣 | 固定金额折扣 |
| D - 商品数量 | 单件商品 | 多件商品 | - |
| E - 结算方式 | 全款支付 | 部分付款 | - |

### 正交表选择

5因素，其中3因素3水平，2因素2水平。

计算自由度：f = (3-1)×3 + (2-1)×2 = 8

选择正交表 L9(3⁴) 适配，调整为混合水平表：

| 试验号 | A(客户类型) | B(支付方式) | C(折扣类型) | D(商品数量) | E(结算方式) |

| 1 | 1(普通) | 1(现金) | 1(无折扣) | 1(单件) | 1(全款) |
| 2 | 1(普通) | 2(微信) | 2(百分比) | 2(多件) | 2(部分) |
| 3 | 1(普通) | 3(支付宝) | 3(固定金额) | 1(单件) | 1(全款) |
| 4 | 2(VIP) | 1(现金) | 2(百分比) | 1(单件) | 2(部分) |

## 测试要点总结

1. **金额计算**: 商品总价、折扣金额、应付金额计算必须准确
2. **库存联动**: 销售成功后必须正确扣减库存
3. **应收账款**: 部分付款需正确生成应收账款
4. **状态流转**: 订单状态流转需符合业务规则
5. **并发控制**: 多人同时销售需正确处理库存竞争
6. **数据一致性**: 销售单、库存、应收账款数据需保持一致

### 金额计算公式验证

```
商品总价 = Σ(单价 × 数量)
折扣金额 = 商品总价 × 折扣率 或 固定金额
应付金额 = 商品总价 - 折扣金额
欠款金额 = 应付金额 - 实付金额
```
